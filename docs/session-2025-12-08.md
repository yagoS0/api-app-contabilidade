## Banco de dados via Docker
1. `docker volume create pgdata_enviar`
2. `docker run -d --name enviar-pg -e POSTGRES_PASSWORD=senha123 -e POSTGRES_DB=enviar -e POSTGRES_USER=enviar -p 5432:5432 -v pgdata_enviar:/var/lib/postgresql/data postgres:16`
3. Conferir container: `docker ps --filter name=enviar-pg`
4. Acessar psql: `docker exec -it enviar-pg psql -U enviar -d enviar`
5. `.env`: `DATABASE_URL=postgresql://enviar:senha123@localhost:5432/enviar`
6. Migrações: `npm install`, `npx prisma migrate dev --name init`

## Fluxo de autenticação
1. Configurar `.env`:
   ```
   AUTH_USERS=[{"username":"admin","password":"trocar","role":"admin"}]
   JWT_SECRET=trocar-este-segredo
   JWT_EXPIRES_IN=1h
   ```
2. Login:
   ```
   curl -X POST http://localhost:3000/auth/login \
     -H "content-type: application/json" \
     -d '{"username":"admin","password":"trocar"}'
   ```
3. Usar token:
   ```
   curl http://localhost:3000/clients \
     -H "authorization: Bearer <token>"
   ```
4. `x-api-key` continua aceito para scripts/legado.

## (Atual) Gestão de usuários/admin

- Script `npm run prisma:seed` usa `ADMIN_EMAIL`/`ADMIN_PASSWORD` para garantir admin ativo.
- Endpoints:
  - `POST /auth/signup` → cria usuário `pending`.
  - `POST /auth/login` / `GET /auth/me` → fluxo JWT tradicional.
  - `GET /admin/users?status=pending` → painel do admin (JWT obrigatório, role=admin).
  - `PATCH /admin/users/:id/approve|reject` e `DELETE /admin/users/:id`.
- `AuthService` agora busca no banco; `AUTH_USERS` fica apenas como fallback temporário.
